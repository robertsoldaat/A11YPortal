Lesson 12 - Creating an accessible responsive table

A11Y Champions Developer Training

First hour

Previous lesson - Responsive tables

During the previous lesson, we encountered these examples of responsive tables.

  • Example table with horizontal scroll
  • Example table with :before
  • Example table with display:none on divs
  • Comparison table with disappearing columns

As homework for this lesson, you investigated these tables further.

  • What issues did you find?
  • What solutions did you think of for these issues?
  • Did you find other responsive tables besides those mentioned?
  • Did you come up with a perfect accessible and responsive table?
  • If not, what hindered you from doing so?

Second hour

Second break out - An indepth look at two collapsing tables

First break out - Accessible Tables (Good)

During this breakout, use a screenreader on a desktop ( Windows / Mac ) and mobile ( Android / IOS) and listen. Lesson 10 Break Out

Check the following:

  1. With Caption = without Caption
  2. thead / tfoot / tbody = DOM order
  3. First cell TD VS TH = order mismatch VO
  4. TH should not be empty
  5. Scope VS Headers = Listen to the difference
  6. Table headers read twice, why is this and is it handy?
  7. When present title is read NOT text node. Accessible name calculation?!
  8. Footer with Scope VS headers for clear row/col information
  9. Check mobility = responsiveness
  10. Check difference VO in combinations with Chrome VS Safari

Second break out - Accessible Tables (Bad)

During this breakout, fork the Test Table codepen and fix the issues in the bad version and use a screenreader on a desktop(Windows/Mac) and mobile(Android/IOS) to test. Lesson 10 BreakOut part2

Present the following:

  1. What are your findings?
  2. How did you fixed it?
